This is an example of enforcing policies in the browser using OPA. The application mimics a basic corporate Contract Management solution.

This is a Proof-of-Concept demonstraing how Open Policy Agent (OPA) can be used alongside a web app, taking care of policy decisions. A rationale doing this is reuse of Policy-as-Code across frontend, network and backend.

The context for this PoC is a basic Contract Managment system.

Tech stack
  • Angular 15
  • Open Policy Agent, code developed against v0.48
  • OPA running in WebAssembly

There is no login. Instead you pick your user below. You can at any point switch user to verify how the interaction with OPA works out. All queries to OPA are logged in the browser console. The contract data is randomized and changes with page reload.

Start by selecting a user and navigate the top menu.

The code demonstrates how OPA can
  • validate form fields using AsyncValidatorFn
  • perform access control to routes using Route.canMatch
  • perform access control on outgoing http traffic using HttpInterceptor
  • trim UI by showing/hiding/enabling/disabling elements
